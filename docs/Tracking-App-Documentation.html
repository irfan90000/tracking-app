Tracking App - Complete Documentation

Last updated: March 2025

1. What is This Application?

This is a Tracking Code Manager application. It allows users to add, edit, and manage JavaScript tracking snippets (like Google Analytics, Facebook Pixel, or other marketing scripts) that run on customer-facing web pages. The key feature is cookie consent: tracking scripts only execute when the visitor explicitly accepts cookies.

2. Main Purpose

  • Privacy compliance - Scripts run only when visitors consent
  • Centralized management - Add, edit, delete tracking codes in one place
  • User ownership - Each user manages only their own tracking codes

3. Two Main Areas of the App

3.1 Admin Area (Authenticated Users)

Routes under authentication: Dashboard, Tracking Codes, Profile.

  • /dashboard - User dashboard
  • /tracking-codes - List all your tracking codes
  • /tracking-codes/create - Add a new tracking code
  • /tracking-codes/{id}/edit - Edit an existing tracking code

Here you manage your tracking scripts: create new ones, edit names and script content, or delete them. Each tracking code has a Name (descriptive label) and Script (the JavaScript code without <script> tags).

3.2 Customer Area (Public Visitors)

Routes that anyone can visit without logging in:

  • /customer - Default customer page (uses first user's tracking codes for demo)
  • /customer/{userId} - Customer page for a specific user's site

When a visitor opens these pages:

  1. A cookie consent banner appears at the bottom
  2. Two buttons: "Accept all" or "Reject all"
  3. If Accept: All tracking codes belonging to that page's owner are executed
  4. If Reject: No tracking scripts run
  5. The choice is stored in localStorage for 30 days (banner won't show again until expiry)

4. How It Works - Flow Diagram

Step 1: You (logged in) go to Settings > Tracking Codes

Step 2: You click "Add Tracking Code", enter a name (e.g. "Google Analytics") and paste the JavaScript snippet

Step 3: The script is saved to the database, linked to your user account

Step 4: A visitor goes to /customer (or /customer/your-id)

Step 5: Cookie banner appears: "We use cookies for tracking. [Reject all] [Accept all]"

Step 6: If visitor clicks "Accept all" → Your tracking scripts run on the page

Step 7: If visitor clicks "Reject all" → No scripts run; banner disappears and won't show again for 30 days

5. Database Structure

Table Columns
tracking_codes id, user_id, name, script, is_active, created_at, updated_at
  • user_id - Links the tracking code to the user who created it
  • name - Descriptive name (e.g. "Google Analytics")
  • script - The JavaScript code (stored without <script> tags; the app wraps it when executing)
  • is_active - Whether the script is enabled (default: true)

6. Technical Stack

  • Backend: Laravel 11.x
  • Frontend: React 18+ with TypeScript/JavaScript
  • Integration: Inertia.js (Laravel + React)
  • Styling: Tailwind CSS
  • Icons: Lucide React
  • Primary Color: #3B5FFF

7. Key Components

Component Purpose
CookieBanner Shows Accept/Reject buttons on customer pages; hides after user chooses
TrackingScripts Executes tracking code scripts when consent is given
TrackingCodeFormFields Shared form fields (Name, Script) for Create and Edit pages
useCookieConsent Hook that manages consent state (localStorage, 30-day expiry)

8. Routes Summary

Route Purpose
GET / Welcome / Home page
GET /dashboard User dashboard (auth required)
GET /tracking-codes List tracking codes (auth required)
GET /tracking-codes/create Form to add tracking code (auth required)
POST /tracking-codes Save new tracking code (auth required)
GET /tracking-codes/{id}/edit Form to edit tracking code (auth required)
PUT /tracking-codes/{id} Update tracking code (auth required)
DELETE /tracking-codes/{id} Delete tracking code (auth required)
GET /customer Customer page with cookie banner (public)
GET /customer/{userId} Customer page for specific user's scripts (public)

9. Security

  • Only authenticated users can create, edit, or delete tracking codes
  • Users can only edit/delete their own tracking codes (via TrackingCodePolicy)
  • Customer pages are public; no login required

10. How to Use - Quick Start

  1. Register or log in to the application
  2. Go to "Tracking Codes" in the navigation
  3. Click "Add Tracking Code"
  4. Enter a name (e.g. "Google Analytics")
  5. Paste your tracking script (without <script> tags) into the textarea
  6. Click Save
  7. Open /customer in a new tab (or incognito) to see the customer page
  8. Click "Accept all" on the cookie banner - your script will run
